Changelog:

Registro de versionamentov1.3.0 atual

Registro de todas as versões e alterações do firmware e interface web do sistema Analista de Fluxo.

  • v1.3.0 Junho / 2026
    Configuração dinâmica, logs com nível e exportação
    • NOVO Página de Configuração (/configuracao.html) com frequência dos sensores e parâmetros operacionais ajustáveis via web sem regravação do firmware.
    • NOVO Endpoint GET/POST /api/config para leitura e escrita dos parâmetros: lotacao_max, sensor_intervalo_ms, cooldown_ms.
    • NOVO Persistência dos parâmetros configuráveis em /config.json no LittleFS — restaurados automaticamente no boot.
    • NOVO Logs com classificação por nível: ERROR, WARN, INFO.
    • NOVO Endpoint /api/logs/export e botão de download dos logs em .txt na interface.
    • NOVO Página de Changelog acessível pela interface gráfica (/changelog.html).
    • MELHORIA Filtros por nível de log (ERROR / WARN / INFO / Todos) na página de Logs.
    • MELHORIA API de Performance (/api/performance) agora retorna tamanho total e livre da PSRAM quando detectada, além de heap_total.
    • MELHORIA Gerenciamento de energia explicitado como Light Sleep automático via FreeRTOS (esp_sleep_enable_timer_wakeup) + Modem Sleep.
    • MELHORIA Buffer de logs ampliado de 50 para 100 entradas.
    • MELHORIA Navegação atualizada: todas as páginas incluem links para Configuração e Changelog.
  • v1.2.0 Junho / 2026
    Interface de performance, profiling e Wi-Fi dinâmico
    • NOVO Página /performance.html com guia única de performance: CPU, memória heap/flash, tasks FreeRTOS e profiling de 5 funções.
    • NOVO Endpoint /api/performance com métricas em tempo real de uso de CPU, tempos de execução, informações de tasks e estado Wi-Fi.
    • NOVO Profiling assíncrono por hardware timer (esp_timer_get_time()) das funções: verificarPassagem, verificarEncoder, atualizarDisplay, salvarDados, handleClient.
    • NOVO Configuração e troca do ponto de acesso Wi-Fi via endpoint /api/config_wifi sem regravação de firmware.
    • NOVO Página de membros da equipe (/membros.html).
    • MELHORIA Cálculo de uso de CPU separado por core e combinado (Core 0 + Core 1).
    • MELHORIA Modem Sleep ativado (WiFi.setSleep(true)) para redução de consumo energético.
    • CORREÇÃO Conexão Wi-Fi refatorada para modo assíncrono sem bloqueio — elimina uso de while() sem yield durante a conexão.
  • v1.1.0 Maio / 2026
    Multitarefa com FreeRTOS e persistência de dados
    • NOVO Separação em duas tasks FreeRTOS: TaskSensores (Core 1) e loop principal (Core 0).
    • NOVO Persistência de dados em LittleFS (/dados.json) — salvo automaticamente a cada 30 segundos.
    • NOVO Recuperação automática do histórico no boot.
    • NOVO Endpoint /api/graficos com dados por hora (24h) e por dia (30 dias).
    • NOVO Endpoint /api/upload_dados para restaurar backup de dados via POST.
    • MELHORIA Substituição de todos os delay() por vTaskDelay().
    • HARDWARE Interrupção de hardware no encoder KY-040 (attachInterrupt + IRAM_ATTR).
  • v1.0.0 Maio / 2026
    Versão inicial — Sistema Analista de Fluxo
    • NOVO Contagem bidirecional de pessoas com sensores IR E18-D80NK (pinos A e B).
    • NOVO Contagem alternativa via encoder KY-040.
    • NOVO Alertas de áudio via DFPlayer Mini ao atingir 50% e 100% da lotação.
    • NOVO Display OLED SSD1306 com 4 telas rotativas: total atual, entradas/hora, saídas/hora e porcentagem de lotação.
    • NOVO Servidor web embarcado no ESP32 via LittleFS.
    • NOVO Dashboard principal com total de pessoas, entradas/saídas na última hora e hora de pico.
    • NOVO Sincronização de hora via NTP (UTC-3).
    • HARDWARE Plataforma: ESP32 NodeMCU-32S, Arduino C++ com FreeRTOS.
